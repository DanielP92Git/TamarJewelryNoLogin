## Typography

Scope: two canonical weights are defined for NEW elements introduced in this phase. Additional weights (650, 750, 800, 850, 900) exist in `bambaYafa-desktop.css` for pre-existing components and are not introduced by this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / table cell content | 13px | 700 | 1.5 (inherited) |
| Label / meta / badge / card sub-label | 12px | 700 | 1.2 |
| Input / select | 14px | 700 | 1.5 |
| Card title / section heading | 14px | 800 | 1.2 |
| Table header row | 11px | 800 | 1.2 |

Canonical weights for this phase:
- **700** — body text, table cell content, inputs, dates, sizes, badge text
- **800** — labels, headings, table column headers, card sub-labels

Notes:
- Weights 650, 750, 850, 900 appear in the existing stylesheet for pre-existing components. Do not introduce them in any new CSS rules written for this phase.
- Monospace content (filenames, backup keys): `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace` at 12px / weight 700 — use `.mono` class.
- Date/time values displayed in table: 13px, weight 700, `var(--muted)` color — consistent with standard row text.
- No new font sizes beyond those listed. All new text falls into one of the five roles above.

## Component Inventory

Components to add. All are CSS-only additions to `bambaYafa-desktop.css` with JS wired in `BisliView.js`.

### 1. Summary Status Card (`.backup-summary-card`)

Focal point: the "Run Backup Now" button on the right side of the card is the primary focal point. It is the single actionable element on the page above the table and must be visually prominent using `.btn--primary` (accent blue). The status badge serves as secondary information. The timestamp and size values are tertiary.

Layout: horizontal flex row, full width, `padding: 16px`, `border-radius: var(--radius)`, `border: 1px solid var(--border)`, `background: rgba(30, 30, 45, 0.76)`.

Columns (left to right):
- Left block: "Last successful backup" label (12px, weight 800, `var(--muted-2)`) + date/time value (13px, weight 700, `var(--text)`) + file size (12px, weight 800, `var(--muted)`)
- Center: status badge (`.badge--success` or `.badge--danger`)
- Right: "Run Backup Now" button (`.btn.btn--primary`) — margin-left: auto

Button states:
- Default: `.btn.btn--primary` with text "Run Backup Now"
- In-progress: disabled attribute set, content replaced with `<span class="button-spinner"></span> Running...` using existing `.button-spinner` class
- Post-operation: button re-enabled, spinner removed, text restored

### 2. Backup History Table (`.backups-table`)

Use CSS Grid layout consistent with existing `.table` + `.row` pattern. Five columns:

```
Date/Time | Size | Status | Type | Actions
```

Column widths: `2fr 0.75fr 0.9fr 0.9fr 120px`

Row rules:
- All rows: date/time via `formatBackupDate()`, size via `formatBytes()`, status via `statusBadge()`, type via `triggerBadge()`
- Rows with `status === 'success' && trigger !== 'restore'`: show `.btn` "Restore" button in Actions column
- All other rows: empty Actions cell
- Failed rows: status badge uses `.badge--danger`; `error` field rendered as `title` attribute on the badge (native tooltip)
- Header row: 11px, weight 800, `var(--muted-2)`, uppercase

### 3. Restore Confirm Modal (`.restore-confirm-modal`)

A new `<dialog>` class, dark-themed to match the dashboard (not the existing light product preview modal).

```css
dialog.restore-confirm-modal {
  /* follows admin-preview-modal sizing pattern */
  max-width: 480px;
  background: var(--surface);
  border: 1px solid var(--border-2);
  border-radius: var(--radius);
  color: var(--text);
  padding: 0;
}

dialog.restore-confirm-modal::backdrop {
  background: rgba(0, 0, 0, 0.65);
  backdrop-filter: blur(3px);
}
```

Internal layout: header / body / footer using `.modal-header`, `.modal-body`, `.modal-footer` structure — override background to `var(--surface)` / `var(--surface-2)` to match dark theme.

Danger tint for confirm state: add `border-top: 3px solid var(--danger)` on the modal container.

State machine visual summary:

| State | Header bg | Body | Footer |
|-------|-----------|------|--------|
| Confirm | `--surface-2` with danger top border | Warning text + backup info + phrase input | "Don't Restore" (`.btn`) + "Restore Database" (`.btn.btn--danger`, disabled until phrase match) |
| In-progress | `--surface-2` | Centered `.button-spinner` (large: 36px) + "Restoring database..." | No footer; no close button |
| Success | `--surface-2` | "Restore Complete" heading + pre-restore filename + duration | "View Backups" (`.btn.btn--primary`) |
| Error | `--surface-2` with danger top border | Error heading + message + failed step + context | "Close Details" (`.btn`) |

### 4. Sidebar Section Addition

New `nav__section` block below existing "Tools" section in `admin/index.html`:

```html
<div class="nav__section">

  <button type="button" class="nav__item sidebar_backups" data-nav="backups">

      <!-- Inline SVG: database cylinder icon (24x24 viewBox) -->
    </span>
    <span class="nav__text">Backups</span>

```

Icon: inline SVG database cylinder — 3 concentric horizontal ellipses with a vertical rectangle body. `width="18" height="18"`, `fill="currentColor"`, `stroke="none"`. Matches the 18px size used on other nav icons.

## Interaction States

| Component | State | Visual Treatment |
|-----------|-------|-----------------|
| "Run Backup Now" button | Default | `.btn.btn--primary` — blue background, white text |
| "Run Backup Now" button | Loading | `disabled` attr + `.button-spinner` (20px) + "Running..." text; opacity unchanged (no dimming per `.btn--primary:disabled` rule) |
| "Run Backup Now" button | Re-enabled after 409 | Re-enable after toast shown (short delay ~5s so user can retry); do not leave permanently disabled |
| Restore button (row) | Default | `.btn` — neutral dark background |
| Restore button (row) | Hover | translateY(-1px), lighter background per `.btn:hover` |
| "RESTORE" phrase input | Empty / wrong | Submit button has `disabled` attribute |
| "RESTORE" phrase input | Exact match "RESTORE" | Submit button `disabled` attribute removed |
| Modal backdrop | Confirm state | Click dismisses modal (standard dialog behavior) |
| Modal backdrop | In-progress state | Click does NOT dismiss — `handleBackdropClick` listener removed |
| Modal ESC key | Confirm state | Closes modal |
| Modal ESC key | In-progress state | `e.preventDefault()` on `cancel` event — ESC suppressed |
| Modal ESC key | Success/Error state | Listener removed — ESC works again |
| "View Backups" button | Success state | Closes dialog + calls `refreshBackupTable()` |

## Data Formatting

Formatting functions to implement (no external libraries):

| Data | Format | Rule |
|------|--------|------|
| Backup timestamp | `Jan 15, 2026, 03:00 AM` | `new Intl.DateTimeFormat(navigator.language, { year: 'numeric', month: 'short', day: 'numeric', hour: '2-digit', minute: '2-digit' }).format(new Date(isoString))` |
| File size | `2.4 MB` / `512 KB` / `—` | `bytes < 1024 → '{n} B'`, `< 1048576 → '{n} KB'`, else `'{n} MB'`; null → `'—'` |
| Duration | `3s` | `Math.round(totalMs / 1000) + 's'` |
| Pre-restore filename | truncated mono | `.mono` class, `max-width: 280px`, `overflow: hidden`, `text-overflow: ellipsis`, `white-space: nowrap` |
| Null fields (size, status) | `—` | Em-dash for all missing/null/undefined cell values |
