<objective>
Make hreflang tags conditional on Hebrew translation existence to prevent identical content across language variants.

Purpose: When Hebrew translations are missing, both /en/ and /he/ product URLs show identical English content. Google treats this as duplicate content and may ignore hreflang signals. This plan makes hreflang bidirectional only when Hebrew translations actually exist, and uses canonical-to-English for untranslated products. The sitemap is also updated to reflect translation status.

Output: Conditional hreflang in meta-tags.ejs, hasHebrewTranslation flag from ssrDynamic.js, conditional sitemap hreflang
</objective>

2. In the product loop (starting at line 100), add conditional hreflang based on translation status. Replace the current unconditional hreflang block:

```javascript
for (const product of products) {
  const imageUrl = product.images?.[0]?.publicDesktop || product.mainImage?.publicDesktop;

  // Check if Hebrew translation exists
  const hasHebrewTranslation = Boolean(product.name_he && product.description_he);

  // Build hreflang links conditionally
  const hreflangLinks = hasHebrewTranslation
    ? [
        { lang: 'en', url: `/en/product/${product.slug}` },
        { lang: 'he', url: `/he/product/${product.slug}` },
        { lang: 'x-default', url: `/en/product/${product.slug}` }
      ]
    : [
        { lang: 'en', url: `/en/product/${product.slug}` },
        { lang: 'x-default', url: `/en/product/${product.slug}` }
      ];

  const img = imageUrl ? [{ url: imageUrl, caption: product.name }] : [];
  const lastmod = product.date ? new Date(product.date).toISOString() : undefined;

  // English version (always included)
  links.push({
    url: `/en/product/${product.slug}`,
    changefreq: 'weekly',
    priority: 0.7,
    lastmod,
    img,
    links: hreflangLinks
  });

  // Hebrew version: only include if translation exists
  if (hasHebrewTranslation) {
    links.push({
      url: `/he/product/${product.slug}`,
      changefreq: 'weekly',
      priority: 0.7,
      lastmod,
      img,
      links: hreflangLinks
    });
  }
}
```

**Key changes:**
- Untranslated products only appear as `/en/product/slug` in sitemap (not `/he/product/slug`)
- Hreflang for untranslated products only includes English + x-default
- Translated products retain full bidirectional hreflang and both language URLs
- Static pages and category pages are NOT changed (always bidirectional, translations are guaranteed)

Do NOT change the static pages or category sections -- those always have content in both languages because their content comes from metaConfig (hardcoded bilingual strings), not from product database fields.
