**1. Update getTokenFromRequest in backend/middleware/auth.js** to also check `req.query.token` as a fallback. Current code checks `req.header('auth-token')` then `req.header('authorization')`. Add a third fallback at the end:

```javascript
function getTokenFromRequest(req) {
  const direct = req.header('auth-token');
  if (direct) return direct;
  const auth = req.header('authorization');
  if (auth && typeof auth === 'string') {
    const parts = auth.split(' ');
    if (parts.length === 2 && /^Bearer$/i.test(parts[0])) return parts[1];
  }
  // Fallback: query parameter (for EventSource which can't send headers)
  if (req.query && req.query.token) return req.query.token;
  return null;
}
```

This is safe because the token is still validated by jwt.verify() in fetchUser. The query param approach is only needed for SSE/EventSource connections.
