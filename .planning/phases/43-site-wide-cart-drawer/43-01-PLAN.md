## Trust Boundaries

| Boundary | Description |

| URL language segment → checkout CTA href | `urlLang` is derived server-side from the route; the CTA points at `/{urlLang}/cart`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | Tampering/Injection (XSS) | `#tk-drawer-body` line rendering | mitigate | This plan only ships a static container + empty-state; NO user data is interpolated via EJS into line items. Dynamic line nodes are built in Plan 03 using DOM `textContent` (never `innerHTML`). EJS here uses only literal bilingual strings + server-trusted `urlLang`. |
| T-43-02 | Tampering (open redirect / path injection) | `data-cart-url="/<%= _urlLang %>/cart"` | mitigate | `_urlLang` is constrained to `'en'`/`'he'` (derived from `lang === 'heb' ? 'he' : 'en'`), never free user input, so the CTA target cannot be redirected off-path. Plan 03 re-validates the client-side language→urlLang map against the `'en'`/`'he'` allowlist. |
| T-43-03 | Information disclosure | empty-state / static labels | accept | Only static marketing copy; no PII or cart data rendered server-side. |
