**Step 2: Create `backend/cache/translationCache.js`**

Follow the exact pattern from `backend/cache/pageCache.js`. Create a dedicated node-cache instance:

**Step 3: Create `backend/services/translationService.js`**

Follow the service pattern from `backend/services/exchangeRateService.js` (external API integration with error handling). The service must:

**Initialization:**
- Import `{ TranslationServiceClient }` from `@google-cloud/translate` (v3 namespace: `require('@google-cloud/translate').v3`)
- Import `{ translationCache }` from `../cache/translationCache`
- Initialize `translationClient = new TranslationServiceClient()` — SDK auto-reads `GOOGLE_APPLICATION_CREDENTIALS` env var
- Read `projectId` from `process.env.GOOGLE_CLOUD_PROJECT_ID`
- **Graceful degradation:** If GOOGLE_APPLICATION_CREDENTIALS or GOOGLE_CLOUD_PROJECT_ID are not set, log a warning at module load time but do NOT throw — let the error surface when translateText is actually called

**`getCacheKey(text, sourceLang, targetLang)` function:**
- Returns `${sourceLang}-${targetLang}:${text}` — includes both source and target to prevent bidirectional cache collisions (per research Pitfall 4)

**`retryWithBackoff(fn, maxAttempts = 3, baseDelay = 1000)` function:**
- Retry loop with exponential backoff and jitter
- Only retry transient errors: codes 429, 500, 503, and network errors (ECONNRESET, ETIMEDOUT)
- Do NOT retry 400, 401, 403 (non-transient — per research anti-patterns)
- Backoff formula: `min((2^attempt + random(0-1)) * baseDelay, 30000)` — cap at 30 seconds
- Log warning on each retry with attempt number, delay, and error message

**`translateText(text, targetLang, sourceLang = null)` function:**
- Validate: text must be non-empty string, targetLang must be 'en' or 'he'
- Trim input text
- Check cache first using getCacheKey — return cached result immediately if found
- Wrap API call in retryWithBackoff:
  - Build request: `{ parent: projects/${projectId}/locations/global, contents: [trimmedText], targetLanguageCode: targetLang, mimeType: 'text/plain' }`
  - Add `sourceLanguageCode` only if sourceLang provided (otherwise auto-detect)
  - Call `translationClient.translateText(request)` — destructure `[response]`
  - Validate response has translations array with at least one entry
  - Build result: `{ translatedText, detectedSourceLanguage }` (include detectedSourceLanguage only if present)
  - Cache result with getCacheKey
  - Return result
- On error: log and re-throw with clear message (distinguish API errors from network errors)

**`translateProductFields(fields)` function:**
- Accepts object with optional: `name_en, name_he, description_en, description_he`
- Auto-detect direction per field pair:
  - If `name_en` exists and `name_he` is empty/missing -> translate name_en to Hebrew
  - If `name_he` exists and `name_en` is empty/missing -> translate name_he to English
  - Same logic for description pair
- Return object with only the newly translated fields (e.g., `{ name_he: "..." }`)
- On error: log and re-throw (caller handles graceful fallback)

**`clearCache()` function:**
- Calls `translationCache.flushAll()` — useful for testing or manual cache reset

**Module exports:** `{ translateText, translateProductFields, clearCache, getCacheKey }`
