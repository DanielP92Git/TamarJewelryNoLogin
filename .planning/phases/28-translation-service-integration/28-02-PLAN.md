**Part A: Add POST /admin/translate/bulk endpoint to index.js**

Add immediately after the `/admin/translate` endpoint. Same middleware chain:

```javascript
// Bulk translation with SSE progress streaming
app.post(
  '/admin/translate/bulk',
  adminRateLimiter,
  fetchUser,
  requireAdmin,
  async (req, res) => { ... }
);
```

**Endpoint logic:**

1. **Set SSE headers:**
   ```javascript
   res.setHeader('Content-Type', 'text/event-stream');
   res.setHeader('Cache-Control', 'no-cache');
   res.setHeader('Connection', 'keep-alive');
   res.flushHeaders();
   ```

2. **Create `sendEvent(event, data)` helper:**
   ```javascript
   function sendEvent(event, data) {
     res.write(`event: ${event}\n`);
     res.write(`data: ${JSON.stringify(data)}\n\n`);
   }
   ```

3. **Handle client disconnect** (for cancellation support — per locked decision "Cancellable mid-operation"):
   ```javascript
   let cancelled = false;
   req.on('close', () => { cancelled = true; });
   ```

4. **Query products needing translation:**
   Use `Product.find()` with `$or` conditions to find products where:
   - `name_en` has content but `name_he` is empty (or vice versa)
   - `description_en` has content but `description_he` is empty (or vice versa)

   Also check for products where legacy `name` has content but `name_en` is empty (pre-migration products).

   Use `.lean()` for performance.

   The query should be:
   ```javascript
   const products = await Product.find({
     $or: [
       { name_en: { $exists: true, $ne: '' }, $or: [{ name_he: '' }, { name_he: { $exists: false } }] },
       { name_he: { $exists: true, $ne: '' }, $or: [{ name_en: '' }, { name_en: { $exists: false } }] },
       { description_en: { $exists: true, $ne: '' }, $or: [{ description_he: '' }, { description_he: { $exists: false } }] },
       { description_he: { $exists: true, $ne: '' }, $or: [{ description_en: '' }, { description_en: { $exists: false } }] },
     ],
   }).lean();
   ```

   Note: Simplify the query if the nested `$or` inside `$or` is problematic. A simpler approach:
   ```javascript
   const allProducts = await Product.find({}).lean();
   const products = allProducts.filter(p => {
     const needsNameHe = p.name_en && !p.name_he;
     const needsNameEn = p.name_he && !p.name_en;
     const needsDescHe = p.description_en && !p.description_he;
     const needsDescEn = p.description_he && !p.description_en;
     return needsNameHe || needsNameEn || needsDescHe || needsDescEn;
   });
   ```
   Use whichever approach is cleaner. With ~94 products total, loading all and filtering in JS is fine.

5. **Send initial count event:** `sendEvent('start', { total: products.length })`

6. **If no products need translation:** Send `sendEvent('complete', { translated: 0, failed: 0, skipped: 0, failedProducts: [] })`, call `res.end()`, and return.

7. **Process each product in a loop:**
   - Check `if (cancelled) break;` at start of each iteration (cancellation support)
   - Send progress event: `sendEvent('progress', { current: i+1, total, productName: product.name || product.name_en || product.name_he, productId: product.id })`
   - Build fields object for `translateProductFields()`: `{ name_en, name_he, description_en, description_he }` from product
   - Call `translateProductFields(fields)` in try/catch
   - If translations returned (non-empty object): save to DB with `Product.updateOne({ _id: product._id }, { $set: translations })` — this is save-as-you-go per locked decision
   - Send success event: `sendEvent('success', { productId: product.id, translations })`
   - Increment translated counter
   - If no translations needed (all fields already filled): increment skipped counter
   - On per-product error: increment failed counter, push to failedProducts array `{ id, name, error: error.message }`, send error event `sendEvent('error', { productId: product.id, productName: ..., error: error.message })`, **continue** (per locked decision: "Continue on failure")
   - **Rate limiting delay:** `await new Promise(resolve => setTimeout(resolve, 100))` — 100ms between products, well under 6000 requests/min limit
   - **SSE keepalive:** Every 30 seconds send `res.write(': keepalive\n\n')` to prevent proxy timeouts (per research Pitfall 5). Track last keepalive time.

8. **Send completion event:** `sendEvent('complete', { translated, failed, skipped, failedProducts })`

9. **Call `res.end()`**

10. **Wrap entire handler in try/catch** — on outer error, send error event and end response.
