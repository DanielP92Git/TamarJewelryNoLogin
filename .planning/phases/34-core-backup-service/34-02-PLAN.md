MOCKING STRATEGY:
The key challenge is mocking child_process.spawn and AWS SDK S3 without ESM/CJS boundary issues. Use vi.mock for both modules.

For child_process.spawn: Use vi.mock('child_process') and make spawn return a mock EventEmitter with stdout/stderr as Readable-like objects:
```javascript
import { vi, describe, it, expect, beforeEach, afterEach } from 'vitest';
import { EventEmitter } from 'events';

// Mock child_process.spawn
vi.mock('child_process', () => ({
  spawn: vi.fn(),
}));

// Mock aws-sdk
vi.mock('aws-sdk', () => {
  const mockPutObject = vi.fn().mockReturnValue({ promise: vi.fn().mockResolvedValue({}) });
  const mockListObjectsV2 = vi.fn().mockReturnValue({ promise: vi.fn().mockResolvedValue({ Contents: [] }) });
  const mockDeleteObjects = vi.fn().mockReturnValue({ promise: vi.fn().mockResolvedValue({ Errors: [] }) });
  const MockS3 = vi.fn().mockImplementation(() => ({
    putObject: mockPutObject,
    listObjectsV2: mockListObjectsV2,
    deleteObjects: mockDeleteObjects,
  }));
  return {
    default: { S3: MockS3, Endpoint: vi.fn(url => ({ href: url })) },
    S3: MockS3,
    Endpoint: vi.fn(url => ({ href: url })),
  };
});
```

Helper to create a mock child process:
```javascript
function createMockChild() {
  const child = new EventEmitter();
  child.stdout = new EventEmitter();
  child.stderr = new EventEmitter();
  return child;
}
```
