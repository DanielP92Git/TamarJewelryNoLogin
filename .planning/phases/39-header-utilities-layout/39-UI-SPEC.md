## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Cart icon padding; badge x-padding; currency select padding top/bottom |
| sm | 8px | Flags gap (`.tk-lang`); badge offset from icon edge; currency select padding-inline-start |
| md | 16px | Not used directly in utils cluster |

Existing gap on `.tk-nav__utils`: **1.4rem (~22.4px)**. Keep as-is — it is established by the prototype and close enough to the lg (24px) token that changing it would cause visual regression.

**Exceptions:**
- Flag icon diameter: **22px** (not on 8-point grid; keep existing — changing it risks regressions in SVG rendering).
- Cart count badge: **18px** min-height (not on 8-point grid; keep existing — changing it risks regressions).
- Cart count badge offset: `top: -6px` / `right: -8px` (keep existing; RTL variant adds `left: -8px`).
- Touch targets: flag icons are 22px — below the 44px minimum for standalone touch targets. Phase 39 scope does not change this; Phase 42 (Mobile Nav) is the correct phase to address mobile touch targets.

## Component Inventory

### 1. Flag Icons (`.flag-icon`)

**Markup:** Two separate `<div class="flag-icon ...">` elements inside `.tk-lang`. Keep existing inline SVG artwork. No emoji, no pill wrapper.

**Sizing:** 22px × 22px. `border-radius: 50%`. `overflow: hidden`.

**States:**

| State | CSS rule | Effect |
|-------|----------|--------|
| Inactive (default) | `.flag-icon` | `opacity: 0.4` |
| Hover | `.flag-icon:hover` | `opacity: 0.85` |
| Active (selected language) | `.flag-icon.selected` | `opacity: 1.0` — NO box-shadow ring |

**Change from current:** Remove `box-shadow: 0 0 0 2px var(--tk-gold)` from `.flag-icon.selected` (and the transparent-nav white-ring variant). Replace with opacity-only treatment.

**Accessibility (icon-only control):** Each flag is an icon-only interactive element that switches language, so it MUST carry a text label fallback for screen readers and pointer tooltips. Apply `aria-label` to each flag element — `aria-label="English"` on the US flag, `aria-label="עברית"` on the IL flag — and mirror it with a matching `title` attribute for hover tooltips. The active flag additionally gets `aria-current="true"`. Confirm these attributes exist in `header.ejs`; add them if absent (opacity is the only visual affordance for the active state, so the accessible name is the sole non-visual cue).

**RTL:** No visual change to flag icon shapes. Cluster reversal (see RTL section) moves flags to the right side of the cluster in RTL.

**Source:** D-01, D-02, D-03 from `39-CONTEXT.md`; `homepage.css` lines 438–453.

### 2. Currency Select (`#currency-desktop`, `.tk-nav__currency-select`)

**Markup:** Keep native `<select>`. Do not replace with custom button+menu. On `/he`, `dir="rtl"` attribute stays on the element.

**Appearance:** `appearance: none` (hide OS native arrow). Custom chevron supplied as CSS `background-image`.

**Custom chevron spec:**
- Asset: inline SVG chevron-down, encoded as data URI in CSS `background-image`
- Color: `--tk-text-muted` #666666 (solid nav), #ffffff (transparent nav)
- Position: `background-position: right 6px center` (LTR); `background-position: left 6px center` (RTL via `[dir="rtl"]`)
- Size: `background-size: 12px 12px`

**Padding (LTR):** `4px 24px 4px 8px` — right side extended to 24px to clear the chevron.
**Padding (RTL):** `4px 8px 4px 24px` — left side extended for chevron on left side.

**Option labels:**
- Default (placeholder): English: `Currency` / Hebrew: `מטבע` (existing)
- USD option: `$ USD` (both languages)
- ILS option: `₪ ILS` (both languages)

**Other styling (keep existing):**
- `border: 1px solid var(--tk-border-strong)` (solid), `rgba(255,255,255,0.6)` (transparent)
- `border-radius: var(--tk-radius-xs)` (4px)
- `font-family: var(--tk-font-body)` (Montserrat / Rubik in RTL)
- `font-size: var(--tk-size-label)` (0.75rem)
- `letter-spacing: var(--tk-track-body)` (0.04em)
- `text-transform: uppercase`
- `background-color: transparent` (solid nav); inherit transparent for hero nav

**Focus state:** Keep browser default `:focus-visible` outline (accessibility). Do not suppress it.

**Dead code to remove:** `.tk-nav__currency` block at `homepage.css` lines 117–123 (unused — markup uses `.tk-nav__currency-select`).

**Source:** D-06, D-07, D-08 from `39-CONTEXT.md`; `homepage.css` lines 117–123, 456–473.

## RTL Layout Contract

**Trigger:** `[dir="rtl"]` on an ancestor element (set server-side by EJS for `/he` routes).

**LTR order (canonical):** Logo (left) | Nav links (center) | [Flags → Currency → Cart] (right)

**RTL mirror order:** [Cart → Currency → Flags] (left) | Nav links (center) | Logo (right)

**Implementation — three CSS rules:**

```
/* 1. Reverse the top-level nav row: logo goes right, utils goes left */
[dir="rtl"] .tk-nav {
  flex-direction: row-reverse;
}

/* 2. Reverse the utils cluster internally: Cart left, Flags right */
[dir="rtl"] .tk-nav__utils {
  flex-direction: row-reverse;
}

/* 3. Flip cart badge to the left side of the icon */
[dir="rtl"] .tk-nav__count {
  right: auto;
  left: -8px;
}
```

**Currency select chevron in RTL:**

```
[dir="rtl"] .tk-nav__currency-select {
  background-position: left 6px center;
  padding: 4px 8px 4px 24px;
}
```

**Constraints:**
- Nav link centering must NOT break — nav links use `display: flex` on `.tk-nav__links`, centering is maintained by `justify-content: space-between` on `.tk-nav` and the logo/utils being `flex: 0 0 auto`.
- Logo position must NOT break — logo is `flex: 0 0 auto`, first DOM child; after `row-reverse` it renders at the right edge, which is the correct RTL position.
- `gap: 1.4rem` on `.tk-nav__utils` is symmetric — keep as-is (D-12).

**Source:** D-09, D-10, D-11, D-12, D-13 from `39-CONTEXT.md`; `tokens.css` `[dir="rtl"]` block.
