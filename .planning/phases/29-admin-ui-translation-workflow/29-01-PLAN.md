**JavaScript (admin/BisliView.js):** Add a `handleTranslateClick` function near the other utility functions (after showInfoToast around line 234). This function:

1. Gets `data-source`, `data-target`, `data-target-lang` from the clicked button
2. Gets source and target input elements by ID
3. Validates source has content — if empty, calls `showErrorToast('Enter text before translating')` and returns
4. If target field already has content, shows `confirm('This field already has content. Overwrite with translation?')` — if user declines, returns
5. Disables button and adds `.is-loading` class
6. Hides any existing error div (ID pattern: `{fieldName}-translate-error` where fieldName is derived from source ID like "name" or "description")
7. Calls `apiFetch('/admin/translate', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer ' + localStorage.getItem('auth-token') }, body: JSON.stringify({ text, targetLang }) })`
8. On success: sets `targetInput.value = data.translatedText`, adds `.field-updated` class to target (remove after 2s with setTimeout), calls `showSuccessToast('Translation complete')`
9. On error: shows inline error in the error div + calls `showErrorToast('Translation failed')`
10. In finally block: ALWAYS re-enables button and removes `.is-loading` class

Also add a `attachTranslateHandlers()` helper function that finds all `.btn-translate` buttons in the current DOM and attaches `handleTranslateClick` as click listener. This will be called after form rendering.

Note: Use `localStorage.getItem('auth-token')` (not 'token') — this matches the existing auth pattern in BisliView.js (see line 4396 for reference).
