# Phase 40.1 — UI Design Contract

> Visual and interaction contract for Phase 40.1: Homepage Featured Products.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Primary purpose of this contract:** Lock the existing `.tk-prod` card / `.tk-prod-grid`
> / `.tk-price` visual treatment so the data-wiring rework does not regress it.
> New UI surface: admin "Featured on Homepage" toggle. New behavior: Sale badge
> auto-logic, card click navigation, dual-price cart drawer re-pricing.

## Spacing Scale

All spacing in this project is rem-based and anchored to the 4px grid via the
existing homepage.css patterns. No explicit `--tk-spacing-*` tokens exist; map
the standard scale to existing usage:

| Token | Value | Existing usage in homepage.css |
|-------|-------|--------------------------------|
| xs | 4px | `--tk-radius-xs`, icon gap |
| sm | 8px | `.tk-line__main { gap: 8px }`, `.tk-line { gap: 16px }` inner |
| md | 16px | `.tk-prod__title-wrap { padding: 1rem 0.75rem 0.5rem }` |
| lg | 24px | `.tk-drawer__head { padding: 24px 28px }` |
| xl | 28px | `.tk-prod-grid { gap: 28px }` — 4px exception: nearest 4px multiple below 32px |
| 2xl | 48px | `.tk-section` top padding (5.5rem ≈ 88px internal, but card content gaps use 48px) |
| 3xl | 64px | major section margins (`margin-top: 5.5rem` ≈ 88px) |

**Exceptions:**
- `.tk-prod-grid` gap is 28px (not 32px) — matches prototype; do not change.
- Card media height is 20rem (320px at 16px base) — fixed; do not change.
- Card width is 20rem (320px) max; grid wraps naturally with `justify-content: center`.

These elements exist and MUST NOT be visually changed by this phase. The data-wiring

```
  .tk-center
    a.tk-btn.tk-btn--primary        "View All Jewelry" / "לכל התכשיטים"
                                    href="/{lang}/necklaces"
```

**Empty-state rule (D-04):** When the SSR query returns 0 featured products, the
entire band (eyebrow + heading + grid + "View All" button) is omitted from `home.ejs`
output. No placeholder text. No empty grid container. The section tag does not render.

**Partial-state rule (D-05):** 1–7 cards render as-is. The flex-wrap grid fills
from left; no minimum card count, no empty-cell fillers.

### Product Card (`.tk-prod`) — Canonical Markup Shape

```html
<div class="tk-prod" data-id="{id}"
     data-usd-price="{Math.round(usd_price)}"
     data-ils-price="{Math.round(ils_price)}"
     data-original-usd-price="{Math.round(original_usd_price || usd_price)}"
     data-original-ils-price="{Math.round(original_ils_price || ils_price)}"
     data-slug="{slug}"
     data-name-en="{name_en}" data-name-he="{name_he}"
     data-images="{JSON.stringify(galleryImages)}">

  <div class="tk-prod__media">
    <img src="{desktopImage}" alt="{productName}" loading="lazy" />
    <!-- Badge — only when discount_percentage > 0 (D-07) -->
    <div class="tk-prod__badge">
      <span class="tk-badge tk-badge--gold">Sale</span>
    </div>
  </div>

  <div class="tk-prod__title-wrap">
    <h3 class="tk-prod__title">{productName}</h3>
  </div>

  <div class="tk-prod__price">
    <!-- priceHTML() equivalent — see Price Tag spec below -->
  </div>

  <button class="tk-prod__add" type="button">Add to Cart</button>
</div>
```

**Card click target:** Clicking anywhere on `.tk-prod` that is NOT the `.tk-prod__add`
button navigates to `/{lang}/product/{slug}`. The card is an implicit link surface —
implement as `cursor: pointer` + JS click handler (existing CSS already sets
`cursor: pointer` on `.tk-prod`). Do NOT change the card to an `<a>` wrapper; use a
delegated click listener that checks `!e.target.closest('.tk-prod__add')`.

**Badge rule (D-07):**
- Show `.tk-prod__badge` only when `discount_percentage > 0` AND `original_*_price > current_price`.
- Badge text: "Sale" (both EN and HE — the word "Sale" is used in both locales in the prototype).
- Badge classes: `tk-badge tk-badge--gold` — gold background (`#c5a572`), `--tk-ink` text.
- Remove "Handmade" badge entirely — it was demo-only cosmetic.
- No admin-configurable badge text.

**Image source field priority (Claude's Discretion):**
```
product.images?.[0]?.publicDesktop
  || product.images?.[0]?.desktop
  || product.mainImage?.publicDesktop
  || product.mainImage?.desktop
  || ''
```
(Mirror of `category.ejs` lines 57–58.)

No `crossorigin="anonymous"` on `<img>` tags (CLAUDE.md rule — breaks CDN images in dev).

### Price Tag (`.tk-price`)

```html
<!-- Regular price -->
<div class="tk-price">
  <span class="tk-price__now">{symbol}{price}</span>
</div>

<!-- Sale price -->
<div class="tk-price">
  <span class="tk-price__now is-sale">{symbol}{price}</span>
  <span class="tk-price__was">{symbol}{originalPrice}</span>
</div>
```

**Currency symbol:**
- USD: `$` prefix
- ILS: `₪` prefix (U+20AA)

**Price rendering from dual data-* attributes:**
- On load: read `localStorage.currency` (`'USD'` or `'ILS'`); use matching `data-usd-price` or `data-ils-price`.
- On `currency-changed` event: swap to new currency's stored value in the data-* attribute — no API call.
- Prices are already `Math.round`ed in SSR; display as integers.

### Cart Drawer — Dual-Price Wiring (D-08)

Cart items must store BOTH prices so the drawer can re-price without re-querying:

```js
// cart item shape (extend current demo object):
{ id, name, image, qty, usd_price, ils_price }
```

On `currency-changed`:
1. Re-render all `.tk-line__total` values using the new currency's stored price.
2. Re-render `#tk-subtotal` with new total.
3. Symbol (`$` / `₪`) updates to match.

Add a new `.card` block to the edit-product form (and add-product form) in

### Card markup pattern

```html
<div class="card">
  <div class="card__header">
    <h3 class="card__title">Homepage Featured</h3>
  </div>
  <div class="card__body" style="display:flex; flex-direction:column; gap:14px;">

    <div class="field">

    <div class="field" id="featured-order-field" style="display:${product.isFeatured ? 'block' : 'none'};">

             value="${product.featuredOrder ?? ''}" min="1" step="1"

      <div class="help">Lower numbers appear first. Leave blank to append to the end of the list.</div>
    </div>

    <!-- Soft note — shown only when total featured count > 8 (injected by JS) -->
    <div class="help" id="featured-count-note" style="display:none; color:#92400e; background:#fef3c7; border-radius:4px; padding:8px 10px;"></div>

  </div>
```

**Interaction — checkbox reveal:**
```js
document.getElementById('is-featured').addEventListener('change', function() {
  document.getElementById('featured-order-field').style.display =
    this.checked ? 'block' : 'none';
});
```

**Soft note copy (D-10):**
```
"{X} products are featured — only the first 8 by Featured Order appear on the homepage."
```
Color: `#92400e` (amber-800) on `#fef3c7` (amber-100) background — matches warning
convention without introducing a new color token.

**Admin form field IDs (must match `addProduct()` DOM reads):**
- `id="is-featured"` → `name="isFeatured"` → `document.getElementById('is-featured').checked`
- `id="featured-order"` → `name="featuredOrder"` → `document.getElementById('featured-order').value`

## Interaction Specifications

### Featured Grid — On Load
1. SSR renders cards with dual-price `data-*` attributes.
2. `homepage.js` `init()` reads `localStorage.currency` (default `'ILS'` if unset, per
   GeoIP convention from Phase 40).
3. Calls `renderProducts()` which reads each `.tk-prod` card's dataset and renders the
   matching currency price via `priceHTML()`.
4. No loading skeleton — cards are server-rendered; JS only re-renders price text.

### Featured Grid — Currency Change
1. `window` fires `currency-changed` CustomEvent (wired in Phase 40 `View.js`).
2. `homepage.js` listener iterates all `.tk-prod` cards, reads `data-usd-price` /
   `data-ils-price` from dataset, updates `.tk-price__now` / `.tk-price__was` text.
3. Cart drawer also re-renders subtotal and line totals using stored dual prices.
4. No page reload, no API call, no loading state.

### Card Interaction
| Trigger | Action |
|---------|--------|
| Click card body (not Add-to-Cart) | Navigate to `/{lang}/product/{slug}` — read slug from `data-slug` attribute |
| Click `.tk-prod__add` | Add to drawer cart; open drawer; show toast (existing behavior) |
| Hover card | `-3px Y translate + shadow-card-hover` (existing CSS; no JS change) |
| Hover card image | `scale(1.04)` (existing CSS; no JS change) |
| Focus card (keyboard) | No special treatment required in this phase |

### Admin Toggle Interaction
| Trigger | Action |
|---------|--------|
| Check "Featured on Homepage" | Reveal `#featured-order-field` immediately (no animation) |
| Uncheck | Hide `#featured-order-field`; clear value (optional UX convenience) |
| Submit form with blank `featuredOrder` | Server assigns `max(featuredOrder) + 1` (pre-save hook) |
| Total featured count > 8 | JS injects soft note text into `#featured-count-note`, sets `display:block` |

## SSR Data Shape — `data-*` Attributes on Featured Cards

Mirrors `category.ejs` line 103. Every `.tk-prod` card rendered by `home.ejs` must carry:

| Attribute | Value | Example |
|-----------|-------|---------|
| `data-id` | `product._id` | `"64a1b2c3..."` |
| `data-usd-price` | `Math.round(product.usd_price \|\| 0)` | `"45"` |
| `data-ils-price` | `Math.round(product.ils_price \|\| 0)` | `"165"` |
| `data-original-usd-price` | `Math.round(product.original_usd_price \|\| product.usd_price \|\| 0)` | `"55"` |
| `data-original-ils-price` | `Math.round(product.original_ils_price \|\| product.ils_price \|\| 0)` | `"200"` |
| `data-slug` | `product.slug` | `"crystal-hoop"` |
| `data-name-en` | `product.name_en \|\| product.name` | `"Crystal Hoop"` |
| `data-name-he` | `product.name_he \|\| ''` | `"חישוק קריסטל"` |
| `data-images` | `JSON.stringify(galleryImages)` | `"[{\"desktop\":\"...\"}]"` |

`homepage.js` reads these via `card.dataset.*` (camelCase: `card.dataset.usdPrice`,
`card.dataset.ilsPrice`, etc.).
