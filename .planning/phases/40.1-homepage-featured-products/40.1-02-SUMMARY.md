## Accomplishments
- Added `const Product = require('../models/Product')` to `ssr.js` and converted `renderHomePage` to an `async` function that queries `{ isFeatured: true, available: { $ne: false }, quantity: { $gt: 0 } }`, sorts by `featuredOrder: 1`, limits to 8, and `.select()`s the public display fields (including the real numeric `id`) under `.lean()`. ## Known Issues / Deferred

### Dual-render gap — homepage.js still overwrites the SSR grid (resolved by Plan 04)
The current `frontend/js/homepage.js` (untracked WIP) `renderProducts()` calls `grid.innerHTML = ''` (line 57) and rebuilds `#tk-prod-grid` from a hardcoded demo `PRODUCTS` array. At runtime this destructively overwrites the new SSR cards this plan ships. Per the CLAUDE.md "SSR + Client Dual-Render" rule, `home.ejs` and `homepage.js` are a paired render — but the `homepage.js` rework (read SSR `data-*` instead of rebuilding, add `currency-changed` hydration) is **explicitly scoped to Plan 04** (this plan's objective: "Produces the SSR DOM that homepage.js (Plan 04) hydrates"). `homepage.js` was intentionally NOT modified here to avoid scope creep into wave 4 and to avoid touching an in-progress untracked file. Plan 04 must remove the `innerHTML = ''` rebuild and hydrate the SSR cards in place.

### "Handmade" still present in home.ejs (not in the featured cards)
The plan's literal acceptance criterion "home.ejs does NOT contain `Handmade`" is over-broad: two pre-existing, legitimate "Handmade" strings remain — the JSON-LD `description` (line 14) and the hero eyebrow "Handmade in Jerusalem" (line 83). Neither is part of the featured card markup. The substantive rule (drop the demo "Handmade" badge from the cards) is satisfied — the featured cards render only the discount-driven "Sale" badge.

### Commit scope note
`home.ejs` was already dirty from the in-progress homepage-redesign milestone before this plan ran. Because git cannot commit a partial file cleanly, the Task 2 commit (`9e80e04`) necessarily captured those pre-existing redesign edits to `home.ejs` alongside the featured-grid changes — this matches the documented "your changes BUILD ON that current state" intent. All other dirty redesign files (footer.ejs, meta-tags.ejs, tokens.css, workshops.js, etc.) were left unstaged and untouched.
