# Phase 40.1 Plan 01: Featured Products Backend Data Layer Summary

**MongoDB schema gains isFeatured/featuredOrder with a global end-of-list auto-order hook and featured_order_idx, plus an admin-gated write path that persists both fields and invalidates the homepage cache.**

## Accomplishments
- Added `isFeatured` (Boolean, default false, indexed) and `featuredOrder` (Number, nullable, min 1) to the Product schema, plus the `featured_order_idx` compound index backing the homepage grid query.
- Added a global (cross-category) pre-save hook that auto-appends a newly-featured product to the end of the featured list (max+10, or 10 if first) and clears `featuredOrder` when a product is un-featured (D-01, D-09).
- Added `invalidateHomePage()` which deletes the four home cache keys (`/en` and `/he` × USD/ILS).
- Wired `/addproduct` and `/updateproduct/:id` to persist the featured fields with NaN-safe coercion behind the existing `requireAdmin` chain, and to call `invalidateHomePage()` after save. Legacy `/updateproduct` left untouched.
