# Phase 40.1 Plan 03: Admin Featured-Products Toggle UI Summary

**Admin add/edit forms gain a "Homepage Featured" card with a checkbox, reveal-on-check Featured Order input, a soft >8 note from the in-memory product list, and submission wiring that sends isFeatured/featuredOrder over both the add-path JSON and the edit-path multipart FormData.**

## Accomplishments
- Inserted an identical "Homepage Featured" `.card` block into both the edit-product (`editProduct()`) and add-product (`loadAddProductsPage()`) form HTML strings, placed after the "Product Identifier" card and before the right-column "Media" card. The edit form interpolates `product.isFeatured` / `product.featuredOrder` for pre-fill; the add form hardcodes the unchecked/blank/hidden default.
- Wired a `change` listener on `#is-featured` in both forms that toggles `#featured-order-field` display between `block` and `none`.
- Implemented the soft non-blocking >8 note (D-10) by reading `state.products.filter(p => p.isFeatured).length` in both forms — no new network call. When the count exceeds 8, `#featured-count-note` is populated and shown.
- ADD submission: `addProduct()` reads `#is-featured`/`#featured-order` (parseInt base-10, null when blank) and adds `isFeatured`/`featuredOrder` to the `productData` JSON posted to `/addproduct`.
- EDIT submission: `updateProduct()` appends `isFeatured` (boolean → stringified `'true'`/`'false'`) and `featuredOrder` (parseInt base-10, empty string when blank) to the multipart `FormData` posted to `/updateproduct/:id`.
