# Phase 40.1 Plan 04: Homepage Featured Grid Hydration Summary

**`homepage.js` now hydrates the Plan-02 SSR featured grid from each card's dual `data-*` attributes in the saved currency, navigates to `/{lang}/product/{slug}` on card click, and stores dual prices on a demo cart drawer that re-prices the grid + drawer live on every `currency-changed` event — the demo `CURRENCY`/`PRODUCTS` and destructive `grid.innerHTML=''` rebuild are gone.**

## Accomplishments
- Removed the hardcoded `var CURRENCY = 'ILS'` and the 8-entry demo `PRODUCTS` array. Added `getCurrency()` (reads `localStorage.getItem('currency')`, normalizes to `usd`/`ils`, defaults `ils`) and made `symbol()`/`money()`/`priceHTML()` accept an explicit currency argument.
- Rewrote `renderProducts()` to HYDRATE the SSR DOM: it selects `#tk-prod-grid .tk-prod`, reads `data-usd-price` / `data-ils-price` / `data-original-*-price`, derives the active-currency price + sale state, and fills only `.tk-prod__price` — it never wipes or rebuilds the grid (CLAUDE.md SSR + Client Dual-Render rule).
- Added delegated card-click navigation (D-06) to `/' + (document.documentElement.lang || 'en') + '/product/' + card.dataset.slug`, guarded so clicks on `.tk-prod__add` are ignored. Bound Add-to-Cart buttons from the SSR DOM. Every grid lookup is null-guarded so the empty-state homepage (no `#tk-prod-grid`, D-04) does not throw.
- Reworked the cart drawer for dual pricing (D-08): `addToCart(card)` reads the card dataset and stores both `usd_price` and `ils_price`; `renderCart()` computes the subtotal and per-line totals in the active currency and renders `money(subtotal, cur)`. The remove-button label is now `Remove Item` (UI-SPEC copy lock).
- Added a `window` `'currency-changed'` listener that validates `detail.currency ∈ {usd, ils}` and re-renders both the grid and the drawer, so switching currency re-prices everything instantly with no reload or API call.

## Threat Surface
All three plan threats were handled as specified:
- **T-40.1-10 (DOM XSS via price/cart innerHTML):** mitigated — price text is built only from `parseInt(...)` integers + a fixed currency symbol; cart-line names originate from `data-*` values already HTML-escaped by EJS at SSR (Plan 02). No free-form strings flow into innerHTML.
- **T-40.1-11 (navigation target tampering):** mitigated — the nav path is `'/' + documentElement.lang + '/product/' + card.dataset.slug`, and the slug is schema-validated (`^[a-z0-9-]+$`) at write time, so it cannot inject a cross-origin or `javascript:` URL.
- **T-40.1-12 (currency-changed spoofing):** accepted — same-origin script; the listener validates `detail.currency ∈ {usd, ils}` and ignores anything else.

No new security surface beyond the plan's threat model.
