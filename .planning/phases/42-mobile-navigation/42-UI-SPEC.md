# Phase 42 — UI Design Contract: Mobile Navigation

> Visual and interaction contract for Phase 42. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase introduces a hamburger button + full-screen overlay mobile menu into the global `tk-nav` chrome.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — vanilla CSS with CSS custom properties (`tokens.css`) |
| Preset | not applicable |
| Component library | none |
| Icon library | inline SVG hand-authored; reuse existing SVG paths from `header.ejs` and `View.js:483` |
| Font (LTR) | Montserrat — loaded via Google Fonts, declared in `tokens.css` as `--tk-font-body` |
| Font (RTL/HE) | Rubik — loaded via Google Fonts, declared in `tokens.css` as `--tk-font-hebrew`; applied via `[dir="rtl"]` remap |

## Spacing Scale

Existing 8-point scale from `tokens.css` — no new spacing tokens introduced this phase.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gap, badge inset padding |
| sm | 8px | Flag-to-flag gap inside overlay controls section |
| md | 16px | Overlay controls section internal padding |
| lg | 24px | Overlay top/bottom section internal padding; controls row gap |
| xl | 32px | Vertical gap between overlay nav links; controls section bottom padding |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- Hamburger button and close ✕ button: **44×44px minimum touch target** each (WCAG 2.5.5; icon SVG is 24×24px centered inside the 44×44px target via `padding: 10px`)
- `--tk-header-height: 80px` is reused as the overlay's top strip height (for the close button area), ensuring visual alignment with the bar

| Role | Size | Weight | Line Height | Letter-spacing |

**Pre-existing context (not declared for this phase, no change):** the nav count badge renders at 0.62rem (~10px) / 600 semibold / line-height 1. It is existing markup and CSS carried into the overlay's cart control unchanged — not a new design choice for this phase.

## Interaction Contract

### Hamburger Button

| Property | Value |
|----------|-------|
| Visibility | `display: none` at `≥800px`; `display: flex` at `<800px` |
| Touch target | 44×44px (`width: 44px; height: 44px; padding: 10px`) |
| Icon | Three horizontal bars, inline SVG `viewBox="0 0 24 24"`, `stroke: currentColor`, `stroke-width: 1.5`, `stroke-linecap: round`, bar lengths 18px, gaps 5px |
| Transparent nav | `color: #fff` (inherits to SVG `currentColor`) |
| Solid nav | `color: var(--tk-text)` (#333333) |
| Placement | Last flex item in the `<header>` bar — appended after `.tk-nav__utils` OR as a direct child of `.tk-nav`; flex layout pushes it to the far end automatically |
| RTL placement | Auto-mirrors to the start (far left) via `dir="rtl"` flex reversal on `<html>` — no explicit CSS override needed |
| Button element | `<button type="button" class="tk-hamburger" aria-expanded="false" aria-controls="tk-mobile-nav" aria-label="Open navigation menu">` |
| On open | `aria-expanded="true"`; `aria-label` switches to "Close navigation menu" (or leave static — close button handles the explicit close action) |
| Style reset | `background: none; border: none; cursor: pointer; display: flex; align-items: center; justify-content: center;` |

### Mobile Nav Overlay

| Property | Value |
|----------|-------|
| Element | `<div id="tk-mobile-nav" role="dialog" aria-modal="true" aria-label="Mobile navigation">` |
| Position | `position: fixed; inset: 0; z-index: 200;` (above header at z-index 100; below future cart drawer at 1100+) |
| Background | `background: #ffffff;` opaque |
| Default state | `opacity: 0; pointer-events: none; visibility: hidden;` |
| Open state class | `.is-open` → `opacity: 1; pointer-events: auto; visibility: visible;` |
| Open animation | `opacity 0→1` + `transform: scale(0.98)→scale(1)` over `0.3s var(--tk-ease)` (`cubic-bezier(0.4, 0, 0.2, 1)`) |
| Close animation | `opacity 1→0` + `transform: scale(1)→scale(0.98)` over `0.2s var(--tk-ease)` |
| CSS transition | `opacity var(--tk-dur-fast) var(--tk-ease), transform var(--tk-dur) var(--tk-ease), visibility 0s linear var(--tk-dur-fast)` (visibility delayed on close) |
| Body scroll lock | `document.body.style.overflow = 'hidden'` on open; restore `''` on close |
| Focus trap | On open: move focus to close ✕ button; on close: return focus to hamburger button |
| Breakpoint | Overlay HTML always rendered in the DOM; JS class toggle gates open/closed state |

### Overlay Layout Specification

```
┌──────────────────────────────────────────┐
│  [close ✕ button] at inset-inline-end    │  height: 80px (= --tk-header-height)
│  20px; top: 50%; transform: translateY   │  close button is 44×44px, vertically centered
│  (-50%)                                  │
├──────────────────────────────────────────┤
│                                          │
│              HOME                        │
│              SHOP                        │
│              JEWELRY WORKSHOP            │  flex: 1; flex-direction: column;
│              ABOUT                       │  justify-content: center; align-items: center;
│              CONTACT ME                  │  gap: 32px (xl)
│                                          │
├──────────────────────────────────────────┤
│  [🇺🇸 flag] [🇮🇱 flag]  [$ USD ▾]      │  height: 80px; display: flex; align-items:
│  border-top: 1px solid --tk-border-light │  center; justify-content: center; gap: 24px
└──────────────────────────────────────────┘
```

**RTL layout:** CSS logical property `inset-inline-end` on close button auto-positions to top-left. Flex `justify-content: center` on links keeps them centered regardless of direction. Controls row auto-mirrors via `dir="rtl"` flex reversal.

### Close ✕ Button

| Property | Value |
|----------|-------|
| Touch target | 44×44px (`width: 44px; height: 44px; padding: 10px`) |
| Position | `position: absolute; inset-inline-end: 20px; top: 50%; transform: translateY(-50%);` within the overlay top strip (80px height) |
| Icon | Fresh 24×24 SVG: two diagonal lines forming ×: `M 6,6 L 18,18 M 18,6 L 6,18`, `stroke: var(--tk-ink)`, `stroke-width: 1.5`, `stroke-linecap: round` |
| Style reset | `position: absolute; background: none; border: none; cursor: pointer; display: flex; align-items: center; justify-content: center;` |
| aria-label | EN: `"Close navigation menu"` / HE: `"סגור תפריט ניווט"` |

### Three Dismissal Methods

| Method | Implementation |
|--------|---------------|
| Hamburger toggle (re-tap) | Click handler on `.tk-hamburger` removes `.is-open` from `#tk-mobile-nav`; removes body overflow lock; updates `aria-expanded` |
| Close ✕ button | Click handler on close button removes `.is-open`; same cleanup |
| Scrim / outside tap | Click on `#tk-mobile-nav` itself (not bubbled from children): `if (e.target === overlayEl) close()` — the full-screen white background acts as the tappable scrim |
| Escape key | Bonus: `keydown` listener for `Escape` closes overlay and returns focus to hamburger |

### Mobile Header Bar: Visibility Contract

| Element | Desktop (≥800px) | Mobile (<800px) |
|---------|------------------|-----------------|
| `.tk-nav__links` nav | visible | hidden (existing rule) |
| `.tk-hamburger` button | `display: none` | `display: flex` |
| Flags (`.tk-lang`) in utils | visible | hidden — shown inside overlay instead |
| Currency select in utils | visible | hidden — shown inside overlay instead |
| Cart icon (`.tk-nav__cart`) | visible | visible |

**DOM strategy note (for planner):** The CONTEXT (D-04) flags that flags/currency must not create duplicate-ID conflicts when relocated. The visual contract above specifies the outcome; the planner chooses the DOM strategy (CSS-only hide/show vs second instance vs JS reparenting), guided by the D-04 pitfall.

### Active Link in Overlay

- Source: existing `.is-active` class on `.tk-nav__link` elements in `header.ejs`
- Overlay link default: `color: var(--tk-ink); opacity: 0.85;`
- Overlay link hover: `color: var(--tk-ink); opacity: 1;`
- Overlay link active (`.is-active`): `color: var(--tk-gold); opacity: 1;` — accent only here
